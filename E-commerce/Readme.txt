Exercise Poo Dart - Contexte E-commerce :

 niveau : simple 

Problème :

Vous êtes chargé de développer une application de gestion de produits pour un site e-commerce. Chaque produit a des caractéristiques spécifiques telles que le nom, le prix, et la quantité en stock. La gestion des produits doit se faire de manière organisée, en utilisant les concepts de base de la programmation orientée objet (POO) en Dart.

Tâches à Réaliser :

1. Création de la Classe Produit :
   - Définissez une classe Produit avec les attributs suivants :
      - nom (String)
      - prix (double)
      - quantiteStock (int)

2. Encapsulation :
   - Encapsulez les attributs de la classe Produit en utilisant des modificateurs d'accès appropriés (public, private).

3. Création d'Objets :
   - Instanciez trois objets de la classe Produit avec des caractéristiques différentes.

4. Affichage des Informations :
   - Ajoutez une méthode à la classe Produit pour afficher les informations de chaque produit.

5. Héritage - Catégories de Produits :
   - Créez une classe CategorieProduit qui hérite de la classe Produit.
   - Ajoutez un nouvel attribut categorie (String) à la classe CategorieProduit.

6. Polymorphisme - Redéfinition de Méthode :
   - Redéfinissez la méthode d'affichage dans la classe CategorieProduit pour inclure également la catégorie.

Instructions :

- Utilisez un éditeur de code Dart en ligne ou votre environnement de développement local pour coder et tester votre solution.
- Soyez prêts à expliquer chaque concept utilisé, en mettant en évidence comment ils ont été appliqués pour résoudre le problème spécifique.
- Lors de la présentation des résultats, exécutez votre code pour montrer la sortie générée par votre solution.
- Expliquez la plus-value de chaque concept dans la résolution précise du problème posé, en mettant en avant la modularité, la réutilisabilité du code, et la clarté du design orienté objet.



Livrables :

1. Code Dart Structuré :
   - Présentez un code Dart clair et bien structuré, mettant en œuvre les concepts de base de la POO.
   - Assurez-vous que le code respecte les conventions de style Dart et utilise des commentaires explicatifs.

2. Capture d'Écran des Tests d'Exécution :
   - Fournissez une capture d'écran montrant l'exécution de votre code sur un éditeur Dart en ligne ou dans votre environnement de développement local.
   - Les résultats affichés doivent illustrer la création des objets, l'encapsulation, l'héritage, et le polymorphisme.

3. Explications Détaillées :
   - Écrivez des explications détaillées pour chaque concept POO utilisé dans votre solution.
   - Expliquez comment chaque concept a été appliqué pour résoudre les différentes tâches.

4. Démo Interactive :
   - Préparez une démo interactive présentant l'utilisation de la classe Produit et la redéfinition de méthode dans la classe CategorieProduit.
   - Montrez comment votre conception orientée objet permet une gestion organisée des produits.

5. Mise en Avant des Plus-Values :
   - Mettez en évidence la plus-value de chaque concept POO dans la résolution précise du problème.
